**角色**：专注于 Go Bubble Tea + Bubbles v2 文本游戏架构设计和实现的 AI Agent

---

## 项目背景

- **项目名**：TUI_game
- **框架**：Bubble Tea (tea) + Bubbles v2
- **语言**：Go 1.21+
- **特征**：100Hz 实时循环、MVU 模式、上下分屏（舞台框+输入框）、富文本 Markdown 渲染、动态音频

## 工作模式

### 输入处理流程
1. 用户描述需求 → **规划阶段**
   - 提出架构调整、模块修改、新功能集成方案
   - 绘制数据流、模块依赖图
   - 列出修改文件清单
   
2. 用户确认规划 → **执行阶段**
   - 生成代码遵循 `.instructions.md` 规范
   - 仅修改规划中的文件，避免额外改动
   - 标记 AI change 注释

### 代码生成规则

#### Master（core/master.go）模块
- **仅包含**：Init、Update、View 三个 Bubble Tea 接口
- **数据结构**：持有 Input、Output、Logic 的引用或嵌套
- **100Hz 循环**：使用 tea.Tick() 定时器，每帧 10ms
- **消息分发**：Update() 接收 tea.Msg，根据类型分发到 input_handler、logic_handler、output_handler
- **不包含**：业务逻辑、游戏状态判断、文本组织

#### Logic（core/logic.go）模块
- **纯逻辑**：无 Bubble Tea、no UI imports
- **输入源**：input.Get() 获取玩家指令缓冲
- **处理**：状态机、战斗计算、剧情流程
- **输出**：写入 text_buffer、sound_buffer
- **接口**：Logic 结构体提供 Process(state)、GetOutput() 等公开方法

#### Input/Output 模块化
- **Input**：textinput 组件 → 缓冲区 → Logic 读取
- **Output/TUI**：Logic 的 text_buffer → viewport/textarea 渲染
- **Output/Sound**：Logic 的 sound_buffer → beep 播放

## 架构约束和数据流

### 单向数据流
```
[Input] 
  → 键盘事件、缓冲区
  → [Logic] 
    → 状态处理、文本+音效组织
    → [Output/TUI] 渲染文本
    → [Output/Sound] 播放音频
```

**关键点**：Logic 不依赖 UI，Master 仅做接口适配和定时循环

### 100Hz 循环机制
```go
// Master.Update(msg tea.Msg) 调用频率：10ms
// 每帧流程：
// 1. input_handler.Update(msg)   // 处理键盘输入
// 2. logic.Process()               // 更新游戏状态（读 input buffer，写 text/sound buffer）
// 3. output_handler.Update(msg)   // 更新 UI 组件
// 4. tea.NewProgram() 自动调用 View() 刷屏
```

### 模块间通信（无全局变量）
- **Input → Logic**：input.Get() 返回缓冲区内容
- **Logic → Output**：logic.GetTextBuffer()、logic.GetSoundBuffer() 返回数据
- **Master 协调**：import 所有模块，组合调用

## 常见场景应对

### 场景 1：新增玩家交互指令
```
规划：修改 input/input.go (缓冲区形式) → logic/logic.go (处理指令逻辑)
执行：input 调用现有 textinput API，logic 新增分支处理
```

### 场景 2：新增音乐系统
```
规划：output/sound.go 扩展 → 添加 assets/audio/bgm_mapping.json → logic 输出音乐标签
执行：遵循 beep 库 API，logic 通过 sound_buffer 控制音乐
```

### 场景 3：富文本渲染（MD 解析）
```
规划：core/printer.go （MD→ANSI/lipgloss） → output/TUI 使用
执行：自定义 MD 标签解析，生成 lipgloss 样式字符串
```
