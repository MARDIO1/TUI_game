# TUI_game 项目编码规范

这是一个 Go TUI 文字游戏项目，使用 Bubble Tea + Bubbles v2 框架。

## 语言和版本约束
- **Go**：1.21+ （参考现有 go.mod）
- **Bubbles**：v2（严格要求，https://github.com/charmbracelet/bubbles v2）
- **Bubble Tea**：最新版本
- **依赖库**：charmbracelet/lipgloss, faiface/beep, 标准库 encoding/json

## 命名约定
- **结构体、接口、函数、常量首字母大写**（导出），如 `Master`, `Init()`, `TEXT_BUFFER`
- **私有变量、局部变量使用下划线命名法**：`input_buffer`, `text_output`, `game_state`
- **包名小写不含下划线**：`core`, `input`, `output`
- **文件名小写**：`master.go`, `logic.go`, `input.go`
- **有物理含义的变量需标注单位**：`delay_ms`, `speed_mps`, `volume_db`

## 模块化要求

### Master（core/master.go）
- **仅包含** Bubble Tea 接口实现：`Init()`, `Update(tea.Msg)`, `View()` string
- **不包含**业务逻辑，禁止在此编写游戏逻辑
- 职责：
  - 初始化子组件（input、output）
  - 100Hz 定时器循环调度
  - 分发消息到相应处理器
  - 必须能无缝解耦到纯逻辑层

### Logic（core/logic.go）
- **纯逻辑处理**，不涉及 Bubble Tea、UI、IO
- 从 input 读取玩家操作（缓冲区）
- 处理游戏状态转移、战斗计算、剧情判断
- 输出文本、音效、音乐信息到指定缓冲区
- **禁止** import "tea" 或任何 UI 库

### Input（input/input.go、input/load.go）
- `input.go`：键盘事件处理、textinput 组件、缓冲区管理
- `load.go`：从 JSON/YAML 读取游戏内容
- 暴露接口：`Get()` 获取实时输入、`Clear()` 清空缓冲、`Focus()/Blur()` 控制焦点

### Output（output/TUI.go、output/sound.go）
- `TUI.go`：从 core/logic 获取文本，前端渲染（Bubbles viewport/textarea）
- `sound.go`：从 core/logic 获取音效/BGM 信息，调用 beep 库播放
- **必须依赖** Bubbles 组件

## 代码风格和最佳实践

### 简洁性
- **用最短代码实现，无复杂算法**
- 不要过度设计模式，避免多层抽象
- 循环、条件、函数体简化到必要程度
- 禁止高级特性（泛型复杂用法、reflect 等，除非必要）

### 注释
- **文件头注释**说明职责：`// Package core: 游戏核心管理` 
- **不要**逐行注释，仅在复杂逻辑添加说明
- **遵循文件头要求**修改，留下 `// AI change:` 标记

### 导入规范
```go
import (
    "standard/lib"              // 标准库
    
    tea "github.com/charmbracelet/bubbletea"
    "github.com/charmbracelet/bubbles/xxx"  // Bubbles v2
    
    "TUI_game/core"             // 本地导入
    "TUI_game/input"
)
```

### 不要做的事情
- ❌ 删除非 AI 生成的文件
- ❌ 自动 git 提交
- ❌ 在根目录生成临时文件（除非明确要求）
- ❌ 添加不必要的测试代码或 try-catch
- ❌ 过度封装（能用数组+枚举就不用结构体）
- ❌ 相同功能的重复文件
- ❌ 在 Master 中混入业务逻辑
